# Hindi Voice Collection Bot

A fully working, end-to-end **Hindi Voice Collection Bot** built with:
- **FastAPI** backend (Python)
- **Strict FSM (State Machine)** to guarantee deterministic business logic and prevent hallucinated workflow steps
- **Groq & Gemini LLMs** for dynamic, natural Hindi NLG (Natural Language Generation)
- **Multi-layered Fallbacks**: Automatic fallback to Rule-Based responses if API quotas are exhausted
- **ElevenLabs TTS** with automatic fallback to **edge-tts** for natural Hindi voice generation
- **Browser Web Speech API** for Hindi STT
- **Vanilla HTML/CSS/JS** frontend

**Live Demo**: 👉 [https://huggingface.co/spaces/dhanesh-96/banking-voice-bot](https://huggingface.co/spaces/dhanesh-96/banking-voice-bot)

---

## Architecture Highlights

- **Deterministic Conversational Flow**: The bot strictly follows a predetermined path (Identity Confirmation → Introduction → Bank Info → Amount Info → Payment Offer → Payment Link). The LLMs only generate natural speech for the *current* stage, while the backend FSM controls the *next* stage.
- **LLM Cascade**: Requests are sent to Groq (`llama-3.3-70b-versatile`). If it fails (e.g., 429 Rate Limit), the system instantly falls back to Google Gemini (`gemini-2.0-flash`).
- **Rule-Based Fallback**: If both LLMs are unavailable, the engine falls back to hardcoded, stage-appropriate Hindi responses, ensuring the call is never dropped.
- **TTS Fallback**: Tries ElevenLabs (Charlie, multilingual Hindi voice). On 422 Quota Exhausted, 429 Rate Limit, or network errors, it instantly fails over to Microsoft edge-tts (free).

## Quick Start (Local Setup)

### 1. Prerequisites

- Python 3.10+
- pip

### 2. Setup

```bash
git clone https://github.com/dhanesh-vaishnav96/Banking-Voice-Bot.git
cd Banking-Voice-Bot/voice_collection_bot

# Create virtual environment (recommended)
python -m venv venv

# Activate virtual environment
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Copy environment config
copy .env.example .env
```

### 3. Configure API Keys
Edit the `.env` file to add your keys:
- `GROQ_API_KEY`
- `GEMINI_API_KEY`
- `ELEVENLABS_API_KEY` (Optional)

### 4. Run the Server

```bash
python -m uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

### 5. Open the Frontend

Open your browser and go to: `http://localhost:8000`

> **Note:** Use **Google Chrome** for best Web Speech API (Microphone) support.

## How to Use

1. **Enter Customer Details** in the left panel (Name, Amount, Bank, Payment Link)
2. Click **▶ Start Conversation** — the bot will speak the greeting in Hindi
3. **Click the mic button** 🎤 and speak your response in Hindi
4. The bot will process your response, determine the FSM transition, generate the response, and reply with voice
5. Continue until the conversation completes (e.g. User agrees to pay, or is the wrong person)
6. Click **📋 View Summary** to view the post-call summary generated by the LLM
7. Click **↺ Reset** to start a new conversation

**Text input** is also available as a fallback — type your response and press Enter.

## Conversation Flow

```
1. Identity Confirmation
   Bot: "Kya meri baat Jitesh Soni ji se ho rahi hai?"
                   ↓ (User says haan/yes/ji)
2. Introduction
   Bot: "Namaste Jitesh ji. Main bank ki collection team se bol raha hoon."
                   ↓ (User acknowledges)
3. Bank Information
   Bot: "Ji. Main ICICI Bank ki taraf se bol raha hoon."
                   ↓ (User acknowledges)
4. Amount Information
   Bot: "Aapke account mein paanch hazaar rupaye ki payment pending hai."
                   ↓ (User acknowledges)
5. Payment Offer
   Bot: "Kya aap abhi payment kar sakte hain?"
                   ↓ (User says haan/bhej do)
6. Payment Link Sent
   Bot: "Bilkul. Main payment link share kar raha hoon..."
                   ↓
                 END
```

**Cross-Cutting Intents (Handled anywhere):**
- **Wrong Person:** Immediately politely ends the call.
- **Already Paid:** Acknowledges and ends the call.
- **Supervisor/Angry:** Routes to supervisor and ends the call.
- **Partial/Delay at Offer:** Asks for exact date, updates system, and ends the call.

## Hugging Face Deployment

This repository is configured to run on Hugging Face Spaces (Docker SDK).
To deploy on Hugging Face:

1. Create a new Space using the Docker SDK template.
2. Push this repository's contents to the Space.
3. In the Space Settings → **Variables and secrets**, add the following Secrets:
   - `GROQ_API_KEY`
   - `GEMINI_API_KEY`
   - `ELEVENLABS_API_KEY`
   - `TTS_PROVIDER=elevenlabs`
   - `LLM_PROVIDER=groq`
